# Задание: ML System Design Document для PrismFlow

## Описание проекта

**Название:** PrismFlow

**Бизнес-цель:** Оптимизировать управление товарными запасами и логистикой путём точного прогнозирования спроса на товары в каждой точке продаж сети супермаркетов.

## Описание текущего процесса

Сейчас заказы товаров формируются вручную на основе интуиции менеджеров → частые ошибки (избыток/дефицит) ведут к потерям из-за просроченных товаров (молоко, хлеб) или упущенной выручки при отсутствии товара (спрос есть, продавать нечего).

Руководство сети хочет автоматизировать процесс управления запасами и логистикой, чтобы уменьшить ошибки и увеличить выручку. В качестве данных предполагается использование накопленной статистики по продажам каждого товара в каждом магазине за последний год.

## Ключевые параметры

| Параметр | Значение |
|----------|----------|
| Количество магазинов | 500 |
| Количество уникальных товаров | 10 000 |
| Количество продаж за последний год по всей сети | 2 670 000 000 |

## Требования к дизайн-документу

### 1. Мастер-документ (README.md)

Дизайн-документ (MLSD) для ML-системы прогнозирования спроса в сети супермаркетов. Формат — Markdown для GitHub.

**Структура документа:**

1. **Цели и предпосылки**
   - Зачем ML? Сравнение с текущим ручным процессом
   - Бизнес-требования и ограничения
   - Функциональные и нефункциональные требования
   - Процесс пилота и критерии успеха
   - Scope MVP vs техдолг

2. **Методология**
   - Тип задачи (time series forecasting на уровне товар×магазин×день)
   - Необходимые данные (история продаж, календарь, промо, погода, остатки)
   - ML-метрики (MAPE, WMAPE, MAE) и связь с бизнес-метриками (% списаний, % out-of-stock)
   - Риски на этапе моделирования

3. **Подготовка пилота**
   - Способ оценки (A/B на 10–20 магазинах, 4–6 недель)
   - Критерии успеха
   - План подготовки

4. **Внедрение в production**
   - Архитектура решения (со ссылками на диаграммы)
   - Инфраструктура и масштабируемость
   - SLA, latency, batch vs online
   - Риски

### 2. BPMN-диаграммы (AS-IS / TO-BE)

Два файла с Mermaid-диаграммами для бизнес-процесса управления запасами:

- `diagrams/bpmn-as-is.md` — процесс AS-IS (до внедрения ML): менеджер магазина → анализирует остатки на глаз → формирует заявку → отправляет в распредцентр → распредцентр комплектует → доставка → приёмка. Акцент на ручных шагах и точках ошибок.
- `diagrams/bpmn-to-be.md` — процесс TO-BE (после ML): ML-система формирует прогноз → автозаявка → менеджер подтверждает/корректирует → распредцентр → доставка → приёмка → фидбэк в модель. Акцент на том, что именно автоматизирует ML.

Формат: Mermaid flowchart. Текстовое описание каждой диаграммы (2–3 абзаца).

### 3. Диаграмма структуры данных (ER)

Файл `diagrams/data-model.md` с ER-диаграммой (Mermaid erDiagram).

Основные сущности: stores, products, sales_history, stock_levels, forecasts, orders, order_items, promotions, calendar.

Текстовое описание: почему именно такая структура, какие данные в каком хранилище (OLTP PostgreSQL, ClickHouse/BigQuery для аналитики, Feature Store для ML-фичей).

### 4. Диаграмма архитектуры системы

Файл `diagrams/architecture.md` с диаграммой архитектуры ML-системы (Mermaid flowchart).

Компоненты: Data Sources, Ingestion Layer (Kafka/CDC), Data Lake/DWH (S3 + ClickHouse), Feature Store, ML Pipeline (Airflow + MLflow), Model Serving (FastAPI), Forecast DB (PostgreSQL), Order Management API, Monitoring (Grafana + Prometheus), Frontend.

### 5. UML-диаграмма компонентов

Файл `diagrams/uml-components.md` с UML-диаграммой компонентов (Mermaid).

Компоненты: DataCollector, ETLPipeline, FeatureEngine, ModelTrainer, ModelRegistry, ForecastService, OrderGenerator, DashboardAPI, WebUI.

### 6. UML-диаграмма последовательности

Файл `diagrams/uml-sequence.md` с двумя UML-диаграммами последовательности (Mermaid sequenceDiagram):

1. Процесс ежедневного формирования прогноза
2. Процесс корректировки заявки менеджером

### 7. UML Use Case (диаграмма прецедентов)

Файл `diagrams/uml-use-cases.md` с диаграммой прецедентов.

Акторы: Менеджер магазина, Категорийный менеджер, Data Scientist, Система.

Прецеденты: просмотр прогноза, корректировка заявки, подтверждение заявки, метрики модели, переобучение, настройка параметров, аналитика, управление промоакциями.
